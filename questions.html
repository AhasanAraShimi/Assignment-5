Question-1: What are the different ways to select an element in the DOM?

There are several way hs to select HTML elements in DOM.They are as follows:

1.document.getElementsByTagName('tag name'):

In this way, all the elements of the particular HTML tag name can be selected. For example, we can select an

tag by writing .getElementsByTagName('h1'), then all the h1 tags of the HTML file will be selected.

2.document.getElementById('id name'):

In this way, a particular element can be selected by using the id of that element.this one is the most used method in the case of selecting elements.

3.document.getElementsByClassName('class name'):

By a particular class name, all the child elements under that class name can be accessed.

4.document.querySelectorAll('#id'):

This is another way to reach an element on the basis of id name.In this way all the child inside that particular parent element can be shown in console as childNodes.

5.document.querySelectorAll('.class name'):

This is another way to reach an element on the basis of class name.In this way all the child inside that particular parent element can be shown in console as childNodes.

6.document.querySelectorAll('tag name'):

Another approach of selecting element on the basis of tag name.

7.document.querySelector('.class / #id child name/tag name')

when we need to get the first child of a particular parent element we can use this method.

Question-2: What is the difference between innerHTML, innerText, and textContent ?

1.innerHTML:

This one shows text inside of an element with a tag name, and it shows comments as well. It provides output even if the parent element is hidden.

2.innerText:

This one shows only text inside of an element, ignores tag names and comments.It doesn't show any output if the display of the parent element is hidden.

3.textContent :

When we want to show the text inside an element, we can use it.The speciality of this method is that it can show the text inside even if the parent element's display is hidden. But it ignores comments and doesn't show the tag names.

Question-3: What is event delegation in the DOM?

DOM event delegation is a method where a parent element works as a representative of the child elements.In this method instead of applying the event handler to every child element, it is applied to the parent element, and a particular property target is used to specify and access child elements. When we click on the targeted child element, the event applied to the parent element executes.For example, if we want to event listener to every li inside ul, we can add event listener to ul, instead of applying it on every li.

Question-4: What is event bubbling in the DOM?

Event bubbling is a procedure where the event handler is applied downwards, but it is executed upwards. For instance, there's a section, inside the section there is a child div and inside the child div there's a p tag. If we add an event listener on the section, div, and p tag, respectively, and click on the p tag, then we will get the output of p tag, div, and section, respectively and it continues like this until the last ancestor on which event handler is added, if there any.
